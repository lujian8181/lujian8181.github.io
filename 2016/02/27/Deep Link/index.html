Deep Link

Wikipedia – Mobile deep linking

In the context of mobile apps, deep linking consists of using a uniform resource identifier (URI) that links to a specific location wihin a mobile app rather than simply launching the app.

Deferred Deep Linking allows users to deep link to content even if the app isn’t already installed. Depending on the mobile device platform, the URI required to trigger the app may be different.

不同移动平台的使用Deep Link的方法不同：
  1. Android devices work through intents
  2. BlackBerry 10 devices works through BB10’s invocation framework
  3. Firefox OS devices works through Web Activities
  4. iOS devices works through the openUrl application method
  5. Windows Phone 8 devices works through the UriMapper class

###Google App Indexing

对于 Android 开发者的益处

推动应用安装量：从 Android 设备进行搜索并且尚未安装您的应用的用户将获得安装应用的机会，从而匹配查询。
促进再次吸引用户：App Indexing 让我们的系统能够推广来自已安装应用的搜索结果并提供增强的显示元素（例如应用图标）。此外，使用 App Indexing API 还可实现自动完成 Android 应用查询的功能。这意味着，在 Google 应用中键入查询的移动用户在该查询与应用内容相匹配时，将会看到您的应用出现在自动完成下拉列表中。

对于 iOS 开发者的益处

促进再次吸引用户：集成 App Indexing SDK 让我们的系统能够推广来自已安装应用的搜索结果并提供增强的显示元素（例如应用图标）。

Android 或 iOS 应用设置 App Indexing 的基本流程如下：

1.在您的移动应用中支持 HTTP 深层链接。
  • Android：向您的清单文件中添加 Intent 过滤器并将您的应用关联到您的网站。
  • iOS 9：支持通用链接并上传 apple-app-site-association 文件。
2. 实现 App Indexing。

使用 App Indexing API for Android。集成适用于 iOS 9 的 App Indexing SDK。

3. 检查您的实现。

Android：通过 Google 搜索控制台的 Search Analytics 选项卡或通过提取 Google 为您的链接附加的引用网站信息，查看您的链接在 Google 搜索结果中的表现形式。iOS：iOS 通用链接的引用网站信息将在未来几个月内提供。
